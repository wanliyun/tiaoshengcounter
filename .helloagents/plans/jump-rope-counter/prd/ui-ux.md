# 跳绳计数器 — UI/UX 设计规范

## 设计风格与美学方向
运动科技感 + 极简克制。深色半透明覆盖层叠加在摄像头预览之上，信息层级清晰聚焦于"当前次数"这一核心指标。避免过度装饰，用数字本身的力量传递运动节奏。

参考方向：运动手表数据面板的克制感 + Apple Watch 健身 App 的信息层级。

## 设计简报
- 界面目的：让用户在跳绳过程中一眼看清当前次数，无需低头或眯眼
- 目标用户与场景：运动中（动态场景），手机放在 1.5~2 米前方
- 主要视口：竖屏手机（360~414dp 宽）
- 情绪方向：专业、轻量、动感
- 记忆点：巨大的中央数字 + 实时 BPM 指示环

## 设计 Token
- 颜色：
  - 背景：透明(覆盖层) / Camera 预览做底
  - 表面：rgba(0,0,0,0.45) 半透明卡片
  - 正文：White #FFFFFF
  - 弱化：rgba(255,255,255,0.65)
  - 强调：运动橙 #FF6B35 / 运动绿 #00D68F（计数中脉冲）
  - 语义色：成功 #00D68F / 暂停 #FFC107 / 异常 #FF4444
- 字体：
  - 计数数字：系统等宽数字 (JetBrains Mono 或 Tabular Lining)
  - 标题/标签：系统默认 (Roboto / 思源黑体)
  - 回退：系统 UI 字体
- 字号：
  - Display (当前次数): 72sp
  - Headline (BPM/时长): 28sp
  - Body (标签): 14sp
  - Caption (提示文字): 12sp
- 间距：4dp 基础单位 (4/8/12/16/24/32/48)
- 圆角：卡片 16dp / 按钮 999dp(胶囊) / 面板 12dp
- 阴影：使用 Material 3 elevation tokens，克制使用

## 布局策略
竖屏单栏布局，无横屏适配（本期不做）。摄像头预览全屏铺满 Safe Area，HUD 覆盖层分为：
- 顶部状态栏区：计时器 + 模式标识
- 中央核心区：当前次数（最大字号）+ BPM 环
- 底部控制区：暂停/继续/结束按钮 + 模式切换

### 状态覆盖

| 状态 | 显示内容 |
|------|----------|
| **IDLE** | 底部显示引导文字 "将手机放置在身前，确保全身入镜"；半透明人体轮廓引导线 |
| **READY** | 人体检测框显示绿色；中央显示 "准备开始" |
| **COUNTING** | 上述完整 HUD；数字周围 BPM 环脉冲动画；每次跳跃数字弹跳 |
| **PAUSED** | 中央数字冻结；底部"继续"/"结束"按钮；半透明遮罩变暗 |
| **LOW_LIGHT** | 顶部黄色警告条 "光线较暗，可能影响识别" |
| **NO_PERSON** | READY 引导线变红闪烁 |

### 历史记录页 (HistoryScreen)
- 顶部：周/月切换标签
- 摘要卡片：本周总次数、总时长、平均 BPM
- 柱状图：最近 7 天或 4 周的数据
- 列表：每次运动记录行（日期、时间、次数、时长、BPM）

### 设置页 (SettingsScreen)
标准设置列表：
- 体重(滑块, 30~150kg)
- 计数灵敏度(三档分段选择)
- 语音播报间隔(分段选择: 关闭/10次/50次/100次)
- 摄像头默认方向(前后切换)

## 动效与过渡策略
- 计数弹跳：数字 scale 从 1.0 → 1.15 → 1.0，duration 200ms，spring 弹性曲线
- BPM 环脉冲：每跳触发环 scale 1.0 → 1.05 → 1.0，150ms
- 状态过渡：IDLE → READY 淡入 300ms；COUNTING → PAUSED 半透明遮罩 400ms
- 页面切换：标准 Material 共享轴过渡
- 所有动效遵循 prefers-reduced-motion

## 暗色模式
- 第一期不做暗色模式切换，默认深色 HUD + 摄像头预览背景
- HUD 始终保持可用对比度（WCAG AA 在深底白字天然满足）

## 设计约束
- 字体数量 ≤2（系统默认 + 等宽数字）
- 每屏 CTA 数量 ≤4
- 强调色不超过 2 个（运动橙 + 运动绿）
- 不使用 emoji 作为图标
- 无装饰性动画，所有动效服务于信息传递
