# 跳绳计数器 — 功能需求

## 功能清单

### Must Have（必须）
- F-001: 实时跳绳计数 — 打开摄像头后自动识别人体姿态并检测跳跃动作，实时显示累计次数
- F-002: 运动计时 — 从第一次检测到跳跃开始自动计时，暂停时停止计时
- F-003: 速度显示(BPM) — 实时计算并显示每分钟跳绳次数（最近 5 秒滑动窗口）
- F-004: 卡路里估算 — 基于体重(默认 60kg)和跳绳次数估算消耗热量（公式: MET×体重kg×时长h, MET=12.0 跳绳中速）
- F-005: 计数模式切换 — 双脚跳模式(默认)和交替跳模式
- F-006: 本地历史记录 — 按日存储每次运动的次数、时长、平均 BPM、卡路里，支持按日/周/月查看
- F-007: 完全离线运行 — 无网络依赖，模型内嵌 APK

### Should Have（应该）
- F-010: 语音播报 — 每达到设定间隔（默认 50 次）TTS 语音播报当前次数
- F-011: 前后摄像头切换 — 支持使用前置或后置摄像头
- F-012: 计数灵敏度调节 — 提供低/中/高 3 档灵敏度以适配不同跳绳风格
- F-013: 人体检测提示 — 未检测到完整人体时在预览上显示引导动画
- F-014: 运动数据导出 — 将历史记录导出为 CSV 文件到本地存储

### Could Have（可以）
- F-020: 目标设定 — 设定每日/单次跳绳目标次数，达成后语音提示
- F-021: 成就系统 — 累计里程碑勋章（如 1000 次、10000 次、连续 7 天）
- F-022: 背景音乐 — 运动时播放本地音乐
- F-023: 暗色模式 — 支持深色主题切换
- F-024: 桌面小组件 — 显示今日跳绳累计次数

### Won't Have（不做）
- F-030: 用户注册/登录系统
- F-031: 云端数据同步
- F-032: 社交分享(朋友圈/排行榜)
- F-033: 多人同时检测计数
- F-034: 视频录制功能
- F-035: 付费订阅/内购

## 业务规则与逻辑
- BR-001: 跳跃判定 — 追踪脚踝关键点(左 index 27, 右 index 28)Y 坐标时间序列，当脚踝 Y 值从高位移至局部最低点（脚着地）时计数+1；两次有效跳跃间隔 ≥150ms；幅度变化需超过阈值（由灵敏度档位决定）
- BR-002: 自动启停 — 检测到首个有效跳跃后自动开始计时和计数；连续 5 秒无有效跳跃则自动暂停
- BR-003: 双脚跳模式 — 取左右脚踝 Y 坐标均值进行判定
- BR-004: 交替跳模式 — 分别追踪左右脚踝；任一脚踝独立通过判定条件即计数+1；单脚重复判定受 150ms 间隔过滤
- BR-005: 卡路里计算 — 使用标准 MET 公式；体重默认 60kg 可在设置中修改；中途暂停不计入有效运动时长
- BR-006: 数据持久化 — 每次运动结束（暂停后手动结束或退出应用）写入 Room 数据库；运动中途应用被 kill 时尽量保存已计数据

## 数据模型与实体关系
```
ExerciseRecord (一次运动记录)
├── id: Long (PK, auto)
├── startTime: Long (开始时间戳 ms)
├── endTime: Long (结束时间戳 ms, 可为空表示进行中)
├── totalCount: Int (总次数)
├── mode: String (双脚跳/交替跳)
├── avgBpm: Float (平均速度)
├── maxBpm: Float (最快速度)
├── calories: Float (卡路里)
└── durationSeconds: Int (有效运动时长)

DailySummary (日汇总, 由 Room Query 实时计算)
├── date: String (yyyy-MM-dd)
├── totalCount: Int
├── totalDurationSeconds: Int
├── exerciseCount: Int (当天运动次数)
└── totalCalories: Float

UserSettings (用户设置, SharedPreferences)
├── weightKg: Float (默认 60)
├── sensitivity: String (低/中/高, 默认中)
├── voiceInterval: Int (播报间隔次数, 默认 50, 0 表示关闭)
├── voiceEnabled: Boolean (默认 true)
└── cameraFacing: String (front/back, 默认 back)
```

## API 契约
本应用为纯本地离线应用，无远程 API。模块间接口在 technical.md 中定义。

## 状态机与流程图

### 运动会话状态机
```
IDLE ──(检测到人体)──> READY ──(首次有效跳跃)──> COUNTING
                                                      │
                                              (连续5秒无跳跃)
                                                      │
                                                      ▼
                                                   PAUSED ──(用户点击结束)──> SAVED
                                                      │
                                              (检测到跳跃恢复)
                                                      │
                                                      ▼
                                                   COUNTING
```
